变量 - variables

每个变量和数据结构在 Kotlin 中都有一个类型。类型很重要，因为它们告诉编译器你允许对该变量或数据结构执行哪些操作。换句话说，它具有哪些函数和属性。

Note:
Every variable and data structure in Kotlin has a type. Types are important because they tell the compiler what you are allowed to do with that variable or data structure. In other words, what functions and properties it has.

在Kotlin 中，变量使用关键字 var 声明。变量可以存储任何类型的值。例如，整数、浮点数、布尔值、字符串等。

集合 -collections

1.List

一组数据，如数字列表或字符串列表，可以存储在集合中。Kotlin 提供了各种集合类型，如 List、Set 和 Map，以帮助你存储和操作数据。
– Lists Ordered collections of items
– Sets Unordered collections of unique items
– Maps Unordered collections of key-value pairs

Note:
When programming, it is useful to be able to group data into structures for later processing. Kotlin provides collections for exactly this purpose.

List可以存储相同类型的元素，并且可以重复。例如，一个数字列表可以包含多个 1。

Note:
Lists store items in the order that they are added, and allow for duplicate items.

当创建列表时，Kotlin 可以推断存储的项目类型。要显式声明类型，请在列表声明后添加类型，并在尖括号 <> 中指定：

Note:
When creating lists, Kotlin can infer the type of items stored. To declare the type explicitly, add the type within angled brackets <> after the list declaration:

2.Set

尽管列表是有序的，并且允许重复项，但集合是无序的，并且只存储唯一项。

Note:
Whereas lists are ordered and allow duplicate items, sets are unordered and only store unique items.

这使得集合成为存储唯一值的好选择。例如，一个水果集合可以包含多个苹果，但只能包含一个香蕉。

Note:
You can see in the set example that because sets only contain unique elements, the duplicate “cherry” item is dropped.

3.Map

Map 是一种键值对集合。您可以通过引用键来访问值。您可以将映射想象成一个食物菜单。您可以通过找到您想要吃的食物（键）来找到价格（值）。

Note:
Maps store items as key-value pairs. You access the value by referencing the key. You can imagine a map like a food menu. You can find the price (value), by finding the food (key) you want to eat. Maps are useful if you want to look up a value without using a numbered index, like in a list.

注意：

  • 每个映射中的键必须是唯一的，这样 Kotlin 才能理解您想要获取哪个值。
  • 映射中可以有重复的值。

Note:
Every key in a map must be unique so that Kotlin can understand which value you want to get.
You can have duplicate values in a map.

创建映射个映射：

  • 使用 mapOf() 创建一个只读映射。
  • 使用 mutableMapOf() 创建一个可变映射。

Note:
To create a map, use mapOf() to create a read-only map, or mutableMapOf() to create a mutable map.

当创建映射时，Kotlin 可以推断存储的项目类型。要显式声明类型，请在映射声明后用尖括号 <> 添加键和值的类型。

Note:
When creating maps, Kotlin can infer the type of items stored. To declare the type explicitly, add the types of the keys and values within angled brackets <> after the map declaration. For example: MutableMap<String, Int>. The keys have type String and the values have type Int.
